# PMO向けドキュメント基盤（Canvas Editor） 設計書

## 1. システム概要
本システムは、プロジェクトマネジメント（PMO）におけるドキュメント作成や情報整理を効率化するための、Reactベースの無限キャンバス型エディタです。
テキスト、テーブル、構成図、画像などを自由な位置に配置・編集でき、WBSやカンバンボードなどのテンプレートから素早くプロジェクト管理空間を構築できます。また、Draw.ioを内包した図表の作成や、OpenAI互換APIによるAIアシスタント機能を統合しています。
さらに **Tauri** を導入することで、Windows等のOSネイティブなデスクトップアプリケーション（.exe）として動作し、超軽量・高速なローカルアプリとしての体験を提供します。

## 2. 技術スタック
* **デスクトップアプリ基盤**: Tauri v2 (Rust + WebView2)
  * `tauri-plugin-dialog`, `tauri-plugin-fs` によるネイティブなファイル保存・書き込み権限（`dialog:default`, `fs:allow-write-file`等）。
* **フロントエンド・フレームワーク**: React 19 (Vite)
* **リッチテキスト・テーブルエディタ**: Tiptap (ProseMirrorベース)
* **ドローイングツール組み込み**: Draw.io Embed (iframe経由でのXML/SVG連携)
* **エクスポート機能**: Rustバックエンド(`rust_xlsxwriter`)によるネイティブなExcel出力、`turndown`によるMarkdown変換
* **AI連携**: OpenAI互換API（カスタムエンドポイント対応）
* **スタイリング**: インラインスタイルおよびCSS/Vanillaベースの独自デザインシステム

## 3. 主要機能

### 3.1 キャンバス操作（無限キャンバス）
* **パン（移動）**: 空きスペースでの左クリック＋ドラッグ、中ボタンクリック、またはAlt+左クリックでキャンバスをパン。
* **ズーム**: マウスホイール、またはツールバーの `+` / `-` ボタンで拡大・縮小。
* **グリッドスナップ**: `GRID_SIZE=40`を基準とした配置・移動により、整然としたレイアウトを維持。
* **A4モード**: 印刷を意識したA4サイズ枠（800×1120）への配置制限モード。

### 3.2 ノード操作
* **テキストノード**: Tiptapを利用したリッチテキストエディタ。文字サイズ、装飾に対応。
* **テーブルノード**: 行列の追加・削除、セルの背景色変更に対応したテーブル。
* **DrawIOノード**: Draw.ioをモーダルで開き、作図を行ってSVGとして埋め込み配置。ダブルクリックでDraw.ioエディタが開き、保存するとキャンバス上にSVG画像として表示。
* **画像ノード**: ファイル選択ダイアログから画像（PNG/JPG/GIF/WebP等）を選択し、Base64としてノードに埋め込み。キャンバス上でドラッグ・リサイズ可能。
* **操作**: ドラッグ＆ドロップでの移動、複数選択、コピー＆ペースト（Ctrl+C/V）、Delete削除、矢印キー移動。
* **リンク機能**: ノードに対して別のページへのリンクを設定でき、画面遷移ボタンを表示。

### 3.3 状態・履歴管理
* **Undo / Redo（元に戻す/やり直し）**: ノードの移動、追加、削除、編集などの状態を `past` / `present` / `future` スタックとして管理し、`Ctrl+Z` / `Ctrl+Y`でシームレスに行き来が可能。
  * ドラッグ移動など高頻度なイベント時は履歴の保存を間引く（`recordHistory=false`）設計としており、パフォーマンス低下やメモリ枯渇を防止する最適化を実施。
* **自動保存連携**: 各ノード上のキーボード入力（Ctrl+S）やフォーカスアウト時に、編集内容がReactのStateへ確実にコミットされるフローを統一。
* **ページ管理**: 複数のキャンバスを「ページ」としてタブ管理でき、動的に追加・名前変更が可能。

### 3.4 テンプレート機能
よく使われるPMO向けフォーマットをワンクリックで新規ページとして展開できます。
* **WBS**: 作業構成要素と担当、スケジュールのテーブル。
* **カンバンボード**: ToDo / In Progress / Doneのリスト構造。
* **運用設計書**: 運用体制、連絡網、障害対応フローのフォーマット。
* **プロジェクト計画書**: スコープ定義やマイルストーンのフォーマット。

### 3.5 AI アシスタント機能
* **OpenAI互換API連携**: ノードごとにAIチャットモーダルを開き、プロンプトに応じた内容の自動生成・プレビュー・適用が可能。テキスト・テーブル・DrawIO XMLのすべてに対応。
* **AI設定画面**: AIアシストモーダル内の「⚙️ 設定」ボタンから以下を設定可能。設定はlocalStorageに永続化。
  * **APIエンドポイント**: OpenAI互換の任意のエンドポイントURL（デフォルト: `https://api.openai.com/v1/chat/completions`）
  * **APIキー**: パスワード入力形式
  * **モデル名**: 使用するLLMモデル名（デフォルト: `gpt-4o-mini`）
* **エラーハンドリング**: APIキー未設定時やAPIエラー時に、チャット内にエラーメッセージを表示。

### 3.6 エクスポート・保存機能
* **JSON保存・読込**: Tauri のネイティブファイルダイアログを通じて、全ページデータを `.json` ファイルとして保存・読込可能。プロジェクトの永続化と復元をサポート。
* **JSONエクスポート**: 現在のノード配置データをJSONとしてクリップボードへコピー。
* **Excelエクスポート (Tauriネイティブ処理)**: 全ページ・全ノードを解析し、座標（x, y）をExcelのセル（列・行）にマッピングして出力。
  * **セルサイズ**: 20pxグリッド基準。Font 11 での可読性を確保。
  * **ノードタイトル**: 各ノードのタイトルが設定されている場合、コンテンツの1行上に太字で自動挿入。
  * **Anchor & Flow 配置ロジック**: 20pxをExcelの1セルとして計算し、要素の「左上座標」に最も近いセルを起点（Anchor）として配置。
  * **テキスト要素（Flow）**: 結合を行わず、起点セルにのみ値を入力（Excel Online AD/AE列結合バグ回避）。
  * **テーブル要素（Island）**: キャンバス上で明示的に結合されている箇所（`colspan`/`rowspan`）のみExcel上で `merge_range` し、表の論理的範囲にのみ罫線を出力。
  * **DrawIO画像**: SVGをCanvas APIでPNGに変換し、Grid位置に画像として埋め込み。data URL形式のSVGも直接処理可能。
  * **画像ノード**: Base64画像データをそのままGrid位置に画像として埋め込み。
  * **列幅統一**: `set_column_range_width_pixels` で全列を一括設定し、Excel Online の列境界バグを回避。

## 4. データ・アーキテクチャ

キャンバスの状態は、最上位の `History` ステートによって管理されています。

```json
{
  "past": [],
  "present": [
    {
      "id": "page_1",
      "name": "Page 1",
      "nodes": [
        {
          "id": "text-001",
          "type": "text | table | file | image",
          "text": "<p>HTML Content</p>",
          "title": "Node Title",
          "linkToPageId": "page_2",
          "imageData": "data:image/png;base64,...",
          "svg": "data:image/svg+xml;base64,...",
          "xml": "<mxGraphModel>...</mxGraphModel>",
          "x": 80,
          "y": 80,
          "width": 280,
          "height": 120
        }
      ]
    }
  ],
  "future": []
}
```

## 5. UIコンポーネント構成
* **`CanvasEditor` (Main Container)**: 全体の状態管理、キーボードイベントのハンドリング、キャンバス描画を行う中心的なコンポーネント。
* **`CanvasNode`**: キャンバス上に配置される各ノードのレンダリングを担う。テキスト/テーブル/DrawIO/画像の4タイプに対応。選択状態、リサイズハンドラ、タイトルバー等のUIをもつ。
* **`TextEditorModal`**: テキスト/テーブル編集時のオーバーレイ表示。Tiptapを使ったリッチな編集インターフェースとツールバーを提供。
* **`DrawioModal`**: 図編集のためのDraw.io iframeオーバーレイモジュール。postMessageを用いたイベント通信でSVG/XMLを保存。
* **`TemplateModal`**: テンプレート選択モーダル。
* **`AiChatModal`**: AIアシストによるチャットインターフェース。プレビュー・適用フロー付き。
* **`AiSettingsModal`**: APIエンドポイント・キー・モデル名の設定モーダル。
* **`JsonPanel`**: 開発者向けのデバッグ/出力用パネル。
